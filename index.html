摘要

Polymarket 是全球最大的去中心化预测市场平台，基于 Polygon 区块链构建，允许用户以 USDC 对真实世界事件的结果进行交易。自2020年创立以来，Polymarket 经历了从小众加密应用到主流信息基础设施的蜕变，尤其在2024年美国大选期间月活交易者突破60万、月交易量超$80亿，成为全球最具影响力的信息定价平台。

2025-2026年，Polymarket 通过推出合规版本 Polymarket US（CFTC 注册的指定合约市场）、与 X（Twitter）达成官方合作、签约 MLB（美国职业棒球大联盟）等举措，正式进入主流化阶段。本报告从定位与愿景、创始人背景、用户规模、技术架构、交互流程、商业模式、营收分析、优劣势、竞品格局、风险与争议、未来展望等维度进行全面深度分析。

一、产品定位与愿景

1.1 核心定位

Polymarket 将自身定位为"全球最大的预测市场"（The World's Largest Prediction Market）——一个用户可以对未来事件结果进行交易的信息市场。

与传统博彩平台的根本区别在于： - 信息发现功能：价格反映市场参与者的集体智慧，具有预测准确性 - 双边市场：用户之间相互交易（P2P），而非与庄家对赌 - 透明结算：基于区块链的链上结算，规则公开透明 - 全球可及：任何持有加密钱包的用户均可参与（受地域限制除外）

1.2 愿景与使命

Polymarket 的核心愿景可概括为：

Shayne Coplan 多次在公开场合表述：传统民调、专家预测、媒体叙事存在系统性偏差，而预测市场通过"真金白银"的激励机制，能更准确地聚合分散信息。Polymarket 的终极目标不仅是一个交易平台，而是成为全社会判断"未来会发生什么"的基础设施。

1.3 产品哲学

  • 信息优于观点：市场价格比任何单一专家更可靠
  • 金融激励驱动真相：当有真金白银投入时，参与者会更认真地搜集和分析信息
  • 去中心化保证公平：链上结算确保规则不可篡改
  • 极简用户体验：降低参与门槛，让"对事件下注"像发推文一样简单

1.4 目标市场

Polymarket 瞄准的不是传统赌博市场，而是一个全新品类——"事件合约"（Event Contracts）市场：

传统市场 Polymarket 替代/颠覆
政治民调 实时选举概率定价
经济预测（美联储加息概率） 事件合约交易
体育博彩（FanDuel/DraftKings） 去中心化体育预测
期权/期货市场（对冲） 事件风险对冲
新闻媒体（信息传递） 概率化信息聚合

5.2 条件代币框架（CTF - Conditional Token Framework）

这是 Polymarket 的核心技术创新之一，源自 Gnosis 团队开发的开源标准：

运作原理： 1. 每个市场创建一对互补的结果代币（Yes Token / No Token） 2. 代币基于 ERC-1155 标准（多代币合约，一个合约管理多种代币） 3. 一对 Yes + No 代币始终可以合成/分解为 $1.00 USDC 4. 代币价格在 $0.00 ~ $1.00 之间波动，代表市场认为该结果发生的概率

示例：

市场："BTC 在2026年底前突破 $100,000？"
- Yes Token 价格 = $0.35（市场认为有35%概率）
- No Token 价格 = $0.65（市场认为有65%概率）
- Yes + No = $1.00（永远成立）

结算时：
- 如果 BTC 确实突破 $100K → Yes Token = $1.00，No Token = $0.00
- 如果未突破 → Yes Token = $0.00，No Token = $1.00

5.3 CLOB（中央限价订单簿）

Polymarket 没有采用 AMM（自动做市商）模式，而是使用了传统金融中常见的订单簿模式：

为什么不用 AMM？ - 二元结果市场（只有 Yes/No）不适合 AMM 的连续曲线定价 - 订单簿允许更精确的价格表达（$0.01 精度） - 减少大额交易的滑点 - 专业做市商可以提供更好的流动性

链下撮合 + 链上结算的设计选择： - 订单提交和撮合在链下进行 → 用户无需为每次挂单/撤单支付 Gas - 成交后在链上结算 → 确保资金安全和透明 - 类似于 dYdX 的混合架构思路

5.4 UMA Optimistic Oracle（乐观预言机）

市场结果的认定使用 UMA 的乐观预言机机制：

流程： 1. 提案阶段：结果发生后，任何人可以提交结果提案（如"特朗普赢了"） 2. 挑战期：设定一段时间窗口（通常数小时），任何人可以质疑提案 3. 无争议：如果无人质疑 → 提案被接受 → 市场结算 4. 有争议：如果有人质疑 → 进入 UMA DVM（争议解决机制）→ 由 UMA 代币持有者投票决定

优势： - 绝大多数市场可以快速结算（无争议情况） - 经济激励确保诚实提案（恶意提案者会被罚款） - 去中心化的争议解决兜底

├── USDC 提取至外部钱包
└── 或继续在平台交易

十五、结论

核心判断

  1. Polymarket 是预测市场赛道无可争议的领导者——在去中心化领域已无真正对手，其挑战主要来自合规赛道的 Kalshi 和潜在入场的科技巨头。

  2. "双线战略"是正确选择——国际版保持加密原生优势和全球覆盖，US版切入万亿级美国体育/事件合约市场，两者互为补充而非替代。

  3. 估值合理但需兑现增长——$80-150亿估值基于预测市场成为主流金融产品的预期。这个预期正在被验证（Kalshi $220亿估值、Meta入场、NYSE投资），但需要持续的用户增长和营收兑现。

  4. 核心风险是监管和信任——内幕交易频发、黑客事件、伦理争议如果持续，可能触发监管收紧或用户流失。Polymarket 需要在"开放性"和"市场诚信"之间找到平衡。

  5. Meta 是最大的潜在威胁——如果 Zuckerberg 认真投入，凭借30亿用户和支付基础设施，可能改变整个竞争格局。Polymarket 的时间窗口可能只有1-2年。

十七、AgentPort 方案：Web3 服务聚合平台

17.1 产品定位

AgentPort 是面向 AI Agent 的 Web3 API 服务聚合与发现平台，支持服务搜索、能力比较、按次调用，并通过 X402 协议实现自动化支付结算。

类比：
├── App Store → 人类发现和使用 App
├── RapidAPI → 开发者发现和调用 API
└── AgentPort → AI Agent 发现和调用 Web3 服务（X402 按次付费）

17.2 X402 协议赋能

X402 是由 Coinbase 推出的开放标准，基于 HTTP 402 状态码实现互联网原生支付。核心特性：

特性 说明
HTTP-native 内嵌于现有 HTTP 请求，无需额外通信
零注册 无需账号、API Key，钱包即身份
按次付费 $0.001 级微支付，用多少付多少
即时结算 链上实时到账，无 T+30 等待
网络无关 支持多链、多币种
信任最小化 Facilitator 无法挪用资金

X402 典型流程：

Agent 请求资源 → Server 返回 HTTP 402 + 支付要求
→ Agent 自动签名稳定币支付 → 附带 PAYMENT-SIGNATURE 重新请求
→ Server 验证支付 → 返回 200 OK + 数据

17.4 Agent 调用完整流程

Agent 想知道"美联储7月会降息吗？"

Step 1: 服务发现
  POST agentport.xyz/discover
  { "query": "美联储降息概率", "category": "prediction" }
  → 返回 3 个可用服务 + 能力/价格/准确率对比

Step 2: Agent 自动选择最优服务
  选中"实时事件概率查询"，$0.001/次

Step 3: X402 付费调用
  GET agentport.xyz/api/predict/fed-rate-cut-july-2026
  → 402 Payment Required
  → Agent 签名 $0.001 USDC 支付
  → 附带 PAYMENT-SIGNATURE 重发请求

Step 4: 获得结果
  200 OK
  {
    "event": "美联储2026年7月降息25bp",
    "probability": 0.38,
    "sources": {"polymarket": 0.36, "kalshi": 0.40, "ai_model": 0.37},
    "confidence": "high",
    "explanation_zh": "近期通胀数据高于预期，降息概率下降3%...",
    "volume_24h": "$12.3M",
    "related_events": [...]
  }

全程：无注册、无 API Key、无预充值、按次即付

17.5 X402 解决了哪些预测市场痛点

Polymarket 痛点 X402 + AgentPort 如何解决
需要注册账号 + API Key X402 无需账号，HTTP 请求自带支付
需要预充值/订阅 按次付费，用多少付多少
单一数据源 聚合多个预测服务，Agent 自动比较选择
英文界面/文档 中文服务描述和能力标注
钱包/USDC 门槛 X402 Agent 侧 1 行代码即可支付
无法评估数据质量 "能力比较"功能提供准确率/延迟/价格对比
API 碎片化 统一服务发现接口

17.6 为什么预测是 AgentPort 最好的切入品类

理由 说明
Agent 天然需要概率 任何决策类 Agent 都需要"未来会怎样"的输入
高频 + 低单价 $0.001/次 × 日万次调用 = 完美的 X402 演示场景
结果可验证 概率准确性可回测 → 服务比较有客观标准
时效性强 实时数据不可缓存 → 必须按次调用
差异化明显 不同服务商的预测能力差异大 → "比较"有意义
华语空白 无现成竞品 → 先发优势明显
